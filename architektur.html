Wie geht eine Änderung live? (Deploy-Prozess)

1. Code ändern → 2. git push auf „main" → 3. Cloudflare baut (2 Min.) → 4. Automatisch live ✅
Der Code liegt auf GitHub (oechteli/klartextliebe). Cloudflare überwacht das Repo und baut bei jedem Push automatisch neu – kein manuelles Hochladen nötig.
Besucher öffnet URL → Cloudflare liefert HTML → Login (Phase 5) → Supabase prüft → Dashboard (Phase 5)
Grün = bereits live   Lila = noch geplant

Cloudflare Access einrichten (interne Seite schützen)

Um klartext-roadmap.pages.dev mit einem E-Mail-Login zu schützen (wie das Familien-Dashboard), einmalig im Cloudflare-Dashboard einrichten:

  1. Cloudflare Dashboard öffnen → one.dash.cloudflare.com (Zero Trust)
  2. Links: Access → Applications → Add an application
  3. Typ wählen: Self-hosted
  4. App-Name: Klartext Roadmap · Domain: klartext-roadmap.pages.dev
  5. Policy hinzufügen: Allow → Emails → monika.oechtering@googlemail.com
  6. Speichern → fertig! Beim nächsten Aufruf erscheint ein Login-Bildschirm.
✅ Kostenlos im Cloudflare Free Plan enthalten · kein eigenes Backend nötig · Login per E-Mail-OTP (kein Passwort)
