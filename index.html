🔄 Wie funktioniert die Website? (Schritt für Schritt)
Deploy-Prozess (wie Änderungen live gehen)
1. Monika / Claude ändert Code → 2. Git Push auf „main" → 3. Cloudflare baut die Seite → 4. In ~2 Minuten live
Der Code liegt auf GitHub (oechteli/klartextliebe). Cloudflare schaut ständig ob sich etwas ändert und baut dann automatisch neu.
Nutzer-Reise (Was passiert wenn jemand die Seite besucht)
Browser öffnet Website → Cloudflare liefert HTML aus → Login (Phase 5) → Supabase prüft Zugang → Dashboard zeigt Pakete
Grau = noch nicht gebaut. Weiß/Grün = bereits live.

⚠️ Wichtige Hinweise

  • Nie npm 11 für package-lock.json nutzen → Cloudflare CI nutzt npm 10. Immer npx npm@10.9.2 install --package-lock-only
  • Vor jedem Push: npm run build lokal testen (Ordner out/ wird erzeugt)
  • Bilder: next/image braucht images.unoptimized: true im statischen Export
  • Windows curl: schlägt fehl – stattdessen PowerShell Invoke-WebRequest nutzen
  • EBUSY-Fehler beim Build: einfach nochmal bauen (Antivirus blockiert kurz)

🚀 Lokale Entwicklung – Schnellstart

cd C:\Users\mail\klartext-liebe
npm install          # einmalig
npm run dev          # startet auf http://localhost:3000
npm run build        # baut statische Seite nach out/
git add -A && git commit -m "..." && git push   # → automatisch live!
